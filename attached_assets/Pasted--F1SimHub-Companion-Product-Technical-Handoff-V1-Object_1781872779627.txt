# F1SimHub Companion – Product & Technical Handoff (V1)

## Objective

Build a lightweight desktop companion application for F1SimHub that automatically captures telemetry from EA Sports F1 25 (and future F1 games) and uploads completed laps to the F1SimHub backend.

The goal is to eliminate manual lap entry and make lap tracking completely automatic.

---

# Vision

The user experience should be:

1. User creates an F1SimHub account.
2. User downloads F1SimHub Companion.
3. User signs in once.
4. User enables UDP telemetry in the F1 game.
5. Every completed lap automatically appears on F1SimHub.

The companion should feel invisible and require no interaction after the initial setup.

# Supported Game Platforms

The companion must work with:

* PC (Steam / EA App / Epic)
* Xbox Series X/S
* PlayStation 5

For console players:

The console broadcasts telemetry over the local network.

The companion runs on any Windows PC or Mac connected to the same Wi-Fi.

The computer does NOT need to run the game.

# Responsibilities of the Companion

The companion should:

* Listen for UDP packets
* Decode EA telemetry packets
* Detect session start
* Detect session end
* Detect lap completion
* Upload lap information
* Retry uploads if offline
* Automatically reconnect
* Auto-update itself
* Run in background
* Launch on startup (optional setting)

# UX Philosophy

The companion should never feel like a separate product.

Users should think:

"My laps automatically appear on F1SimHub."

—not—

"I have to use another telemetry app."

The companion exists solely as an invisible bridge between the game and the F1SimHub platform.

---

# Success Criteria (Version 1)

A user should be able to:

* Install the companion in under 2 minutes.
* Sign in once.
* Enable UDP telemetry in F1 25.
* Complete a lap.
* Refresh F1SimHub.
* See the lap automatically uploaded without any manual input.

If this workflow is seamless, the core objective has been achieved.
