# SIM RACING HQ — Full Build Spec for Replit AI

## Project Overview
Build a personal sim racing companion web app called **Sim Racing HQ**. This is a single-user personal tool to log sessions, track improvement, manage car setups, and study F1 circuits. It should feel like a professional F1 team's internal dashboard — dark, elegant, technical, and precise.

---

## Tech Stack
- **Framework:** React (Vite)
- **Styling:** Pure CSS with CSS variables — NO Tailwind, NO component libraries
- **Charts:** Recharts (npm install recharts)
- **Icons:** Lucide React (npm install lucide-react)
- **Data persistence:** localStorage only — no backend, no auth, no database
- **Routing:** State-based navigation (no React Router) — a single `page` state variable controls which page renders

---

### Design Rules
- Background is near-black `#080808` — everything lives on this
- Cards use `#111111` with `1px solid #1E1E1E` borders
- **Red** (`#E8002D`) is the ONLY accent color for active states, CTAs, highlights
- **Teal** (`#00D2BE`) for lap times and sector data — the classic F1 timing color
- **Yellow** (`#FFF200`) for PB/personal best indicators only
- **NO rounded corners** — everything is sharp/square, like a telemetry display
- A 2px red top border on cards indicates importance or hover
- All text labels in `Orbitron` uppercase with wide letter-spacing
- Nav is a fixed left sidebar, 240px wide
- All buttons are flat, uppercase, letter-spaced — never pill-shaped
- Tables have minimal borders, just bottom borders per row
- Hover states are subtle: `rgba(255,255,255,0.02)` background shift

## Key Functional Requirements

1. **PB auto-detection:** When saving a session, compare bestLap against all previous sessions for the same trackId + car combo. If faster (or first session for that combo), set `isPB: true` on that session object.

2. **Lap time storage:** Store times as strings (e.g. `"1:21.456"`). For chart comparisons and sorting, convert to seconds using a helper: split on `:`, parse minutes and seconds, return `minutes * 60 + seconds`.

3. **Corner table editing:** Clicking any cell in the corners table makes it an editable input. On blur, autosave to localStorage. No save button needed.

4. **Setup compare:** User checks two setup cards, click COMPARE. Render a two-column table. Any value that differs between the two setups gets highlighted with red text on both sides.

5. **Heatmap generation:** On mount, build a map of `date → session count` from sessions array. Generate 365 day cells from today backwards. Assign CSS class based on count: 0 = empty, 1 = l1, 2 = l2, 3+ = l3.

6. **localStorage keys:** Use `sim_sessions`, `sim_track_notes`, `sim_setups` as keys.

7. **ID generation:** Use `crypto.randomUUID()` for all new record IDs.

8. **Empty states:** Every page/section with no data should show a centered empty state with an Orbitron heading and Rajdhani description prompting the user to add data.
